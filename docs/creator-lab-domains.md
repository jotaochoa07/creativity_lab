# Creator Lab — Mapa de dominios (skill tree)

> Referenciado desde `CLAUDE.md`. Este documento es el mapa completo de contenido de Creator Lab. No todos los dominios están en el MVP — ver estado de cada uno.

## El motor genérico: ciclo de entrenamiento

Todos los dominios entrenables comparten **la misma estructura de 8 pasos**. Esto no se construye 20 veces — se construye **una sola vez** como motor reutilizable (`/core/curriculum`) y cada dominio es una configuración de ese motor.

1. **Fundamentos** — aprender el principio
2. **Demostraciones** — ver buenos ejemplos
3. **Prácticas guiadas** — ejercicios concretos
4. **Retos** — subir el nivel de dificultad
5. **Feedback de IA** — crítica estructurada (usa `/core/mentor`)
6. **Repetición deliberada** — volver a practicar el punto débil
7. **Métricas de progreso** — evidencia de mejora (usa `/core/gamification`)
8. **Portafolio** — guardar las mejores versiones (usa `/core/portfolio`)

Esta es la diferencia central del producto frente a una biblioteca de cursos: se premian repeticiones y mejora medible, no consumo pasivo.

## Nota sobre el Dominio 19

"Sistema Operativo del Creador" **no es un dominio entrenable**. Es la capa de workspace que envuelve a todo el producto (segundo cerebro, SOPs, dashboards, calendario editorial, revisión semanal, repositorio de prompts, gestión de proyectos). Vive en `/core/portfolio`, no en el skill tree. Se documenta acá igual para no perder el contenido original.
